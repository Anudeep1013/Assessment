# RAG Pipeline - Retrieval Augmented Generation

A production-ready Python pipeline for document processing, semantic search, and AI-powered result augmentation using embeddings and hybrid search strategies.

## Overview

This project implements a complete Retrieval-Augmented Generation (RAG) pipeline that:

- **Extracts** text and metadata from multiple document formats (PDF, Markdown, plain text)
- **Chunks** documents into semantically coherent segments with configurable overlap
- **Embeds** chunks using transformer models (sentence-transformers)
- **Indexes** embeddings for fast similarity search
- **Searches** using hybrid approach (60% semantic + 40% lexical ranking)
- **Augments** results with AI-generated semantic captions

## Architecture

```
Document Input
    ↓
[Extract] → Parse text & metadata from various formats
    ↓
[Chunk] → Split into overlapping segments (configurable)
    ↓
[Embed] → Generate 384-dim embeddings (sentence-transformers)
    ↓
[Index] → Store in vector index with metadata
    ↓
[Search] → Hybrid search (semantic + lexical)
    ↓
[Caption] → Add AI-generated summaries (mock/OpenAI/Azure)
    ↓
Results with Metadata & Captions
```

## Key Features

### 1. **Multi-Format Document Support**
- PDF files (via pypdf)
- Markdown documents
- Plain text files
- Azure Blob Storage integration

### 2. **Intelligent Chunking**
- Format-aware text splitting
- Configurable chunk size and overlap
- Preserves document structure and metadata

### 3. **Semantic Search**
- 384-dimensional embeddings using sentence-transformers
- Cosine similarity scoring
- Fast vector-based retrieval

### 4. **Hybrid Search**
- 60% semantic relevance (embedding similarity)
- 40% lexical relevance (BM25 ranking)
- Combines strengths of both approaches

### 5. **Semantic Captions**
Three caption generation modes:
- **Mock**: Instant, no API key (for testing)
- **OpenAI**: GPT-3.5-turbo or GPT-4 via OpenAI API
- **Azure OpenAI**: GPT models via Azure endpoints

Automatic fallback to mock mode on API failures.

### 6. **Production-Ready Features**
- Comprehensive logging at all pipeline stages
- Error handling and graceful degradation
- Batch processing with progress indicators
- Index persistence (pickle format)
- Type hints throughout

## Quick Start

### Basic Pipeline Usage

```python
from src.ingest import ETLPipeline

# Initialize pipeline with mock captions (no API key needed)
pipeline = ETLPipeline(
    enable_captions=True,
    caption_provider='mock'  # or 'openai', 'azure'
)

# Ingest documents
pipeline.ingest_from_files([
    ('path/to/doc1.pdf', 'doc1.pdf'),
    ('path/to/doc2.txt', 'doc2.txt')
])

# Search
results = pipeline.search(
    query='machine learning fundamentals',
    search_type='hybrid',  # or 'semantic', 'lexical'
    top_k=5
)

# Access results
for result in results:
    print(f"Score: {result.score:.4f}")
    print(f"Content: {result.content}")
    print(f"Caption: {result.semantic_caption}")
```

### With OpenAI Captions

```python
import os

# Set your OpenAI API key
os.environ['OPENAI_API_KEY'] = 'your-api-key'

# Create pipeline with OpenAI
pipeline = ETLPipeline(
    enable_captions=True,
    caption_provider='openai'  # Uses GPT-3.5-turbo by default
)

# Use as normal - captions will be generated by OpenAI
results = pipeline.search('your query', top_k=5)
```

### Export Results

```python
import json

# Convert results to JSON
results_json = [r.to_dict() for r in results]

# Save to file
with open('results.json', 'w') as f:
    json.dump(results_json, f, indent=2)

# Each result includes:
# - chunk_id: Unique identifier
# - content: Document excerpt
# - source: Document filename
# - combined_score: Hybrid search score (0-1)
# - semantic_score: Embedding similarity
# - lexical_score: BM25 ranking
# - semantic_caption: AI-generated summary
# - metadata: Additional document metadata
```

## Core Modules

### extract.py
Extracts text and metadata from documents.

**Classes:**
- `DocumentExtractor`: Base extractor for text files
- `AzureBlobExtractor`: Extracts from Azure Blob Storage

### chunk.py
Intelligently chunks text into retrieval-friendly segments.

**Classes:**
- `TextChunker`: Format-aware text splitting

### embed.py
Generates semantic embeddings for text chunks.

**Classes:**
- `EmbeddingGenerator`: Wrapper around sentence-transformers

### index.py
Manages vector indexing for fast retrieval.

**Classes:**
- `VectorIndex`: Stores and retrieves embedded chunks

### search.py
Implements hybrid search combining semantic and lexical approaches.

**Classes:**
- `HybridSearchEngine`: Combines semantic + lexical ranking

### caption.py
Generates semantic captions for search results.

**Classes:**
- `CaptionGenerator`: Abstract base
- `MockLLMCaption`: No API calls, instant (for testing)
- `OpenAICaption`: Real captions via OpenAI
- `AzureOpenAICaption`: Real captions via Azure

### ingest.py
Orchestrates the complete ETL pipeline.

**Classes:**
- `ETLPipeline`: End-to-end document processing and search

## Configuration

### Pipeline Parameters

```python
pipeline = ETLPipeline(
    # Chunking
    chunk_size=512,           # Characters per chunk
    chunk_overlap=128,        # Overlap between chunks
    
    # Search
    semantic_weight=0.6,      # Weight for semantic scoring
    lexical_weight=0.4,       # Weight for lexical scoring
    
    # Captions
    enable_captions=True,     # Generate captions for results
    caption_provider='mock',  # 'mock', 'openai', or 'azure'
    caption_max_length=150    # Maximum caption length
)
```

### Search Parameters

```python
results = pipeline.search(
    query='your search query',
    search_type='hybrid',     # 'hybrid', 'semantic', or 'lexical'
    top_k=5,                  # Number of results to return
    filters=None              # Optional metadata filters
)
```

## Output Format

Search results are returned as `SearchResult` objects with the following structure:

```python
class SearchResult:
    chunk_id: str              # Unique chunk identifier
    content: str               # Document excerpt
    source: str                # Source filename
    combined_score: float      # Hybrid search score (0-1)
    semantic_score: float      # Embedding similarity
    lexical_score: float       # BM25 ranking
    metadata: Dict             # Additional metadata
    semantic_caption: str      # AI-generated caption
```

JSON representation:
```json
{
  "chunk_id": "doc1_chunk_0",
  "content": "Machine learning is...",
  "source": "doc1.pdf",
  "combined_score": 0.85,
  "semantic_score": 0.92,
  "lexical_score": 0.76,
  "metadata": {"page": 1, "type": "text"},
  "semantic_caption": "An overview of machine learning concepts"
}
```

## Known Limitations

1. **Index Persistence**: Currently uses pickle format. For production, consider migrating to vector databases (Pinecone, Weaviate, Milvus).

2. **Embedding Model**: Fixed to `sentence-transformers/all-MiniLM-L6-v2` (384 dimensions). Can be extended to support other models.

3. **LLM Providers**: Caption generation supports OpenAI and Azure OpenAI. Can be extended to support other providers (Anthropic, etc.).

4. **Memory**: All indexes stored in memory. Large datasets (>1M chunks) may require database solutions.

5. **Concurrent Access**: Single-threaded processing. For high-concurrency scenarios, consider adding threading/async support.

## Assumptions

- Document encoding is UTF-8
- Chunk sizes are set appropriately for your domain (default 512 chars)
- Search queries are English text (can be extended for multilingual)
- API keys (when used) are valid and have sufficient quota
- Embedding model has sufficient disk space (~500MB for sentence-transformers)

## Performance Characteristics

- **Embedding Generation**: ~100-500 chunks/second (varies by hardware)
- **Search Latency**: <100ms for 10K chunks (semantic)
- **Index Size**: ~1.5KB per 384-dim embedding
- **Memory**: ~150MB for 100K chunks with metadata
